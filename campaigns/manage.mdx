# Campaigns

The MegaSend Campaigns API lets you send WhatsApp template messages in bulk to large recipient audiences. Campaigns are driven by approved WhatsApp message templates and support two audience sources: **Google Sheets** (with column mapping) and **Contact Segments** (saved audience filters from your contacts database). Progress is tracked in real time via per-recipient delivery statuses, aggregate stats, and an audit log.

<Note>
Campaigns are supported on **WhatsApp instances only**. Attempts to create a campaign on Telegram, Messenger, TikTok, or Web instances will return HTTP 400.
</Note>

## Campaign Object

A campaign goes through a defined lifecycle: `draft` → `running`/`scheduled` → `completed` (or `paused`, `cancelled`, `failed`).

| Field | Type | Description |
|---|---|---|
| `id` | UUID | Campaign ID |
| `name` | string | Display name |
| `description` | string \| null | Optional description |
| `template_name` | string | Approved WhatsApp template name |
| `template_language` | string | Template language code (e.g. `he`, `en_US`) |
| `template_id` | string \| null | WhatsApp template ID (optional) |
| `source_type` | string | `google_sheets`, `segment`, `csv_upload`, or `manual` |
| `source_url` | string \| null | Google Sheets URL (sheets source only) |
| `status` | string | Current status — see [Status Values](#status-values) |
| `send_type` | string | `immediate` or `scheduled` |
| `scheduled_at` | datetime \| null | Execution time for scheduled campaigns |
| `total_recipients` | integer | Accepted recipients after validation |
| `messages_sent` | integer | Messages dispatched to WhatsApp |
| `messages_delivered` | integer | Confirmed delivered count |
| `messages_read` | integer | Read receipt count |
| `messages_failed` | integer | Failed send count |
| `throttle_rate` | integer | Messages per minute (auto-optimized unless overridden) |
| `batch_size` | integer | Recipients per Celery batch (auto-optimized) |
| `max_messages` | integer \| null | Budget cap — campaign stops after this many sends |
| `image_url` | string \| null | Static header image URL or Meta media ID |
| `video_url` | string \| null | Static header video URL or Meta media ID |
| `document_url` | string \| null | Static header document URL or Meta media ID |
| `expiration_time_ms` | integer \| null | Unix ms timestamp for Limited-Time Offer countdown |
| `estimated_cost` | float \| null | Pre-send cost estimate |
| `actual_cost` | float \| null | Actual billing cost |
| `shopify_conversions` | integer \| null | Orders attributed to this campaign (Shopify-connected instances only) |
| `shopify_revenue` | float \| null | Revenue attributed to this campaign |
| `shopify_revenue_currency` | string \| null | Currency code for `shopify_revenue` |
| `shopify_conversion_rate` | float \| null | Conversion rate percentage (conversions / messages_sent × 100) |
| `auto_paused` | boolean \| null | `true` if system auto-paused due to error rate |
| `pause_reason` | string \| null | Reason for auto-pause |
| `started_at` | datetime \| null | When the campaign started running |
| `completed_at` | datetime \| null | When the campaign finished |
| `paused_at` | datetime \| null | When the campaign was last paused |
| `created_at` | datetime | Creation timestamp |
| `updated_at` | datetime | Last update timestamp |

### Status Values

| Status | Description |
|---|---|
| `draft` | Created, not yet started |
| `scheduled` | Queued for future execution |
| `running` | Currently sending messages |
| `paused` | Manually or automatically paused |
| `completed` | All recipients processed |
| `cancelled` | Permanently stopped; pending recipients skipped |
| `failed` | Stopped due to a system error |

### Recipient Statuses

Each individual recipient in a campaign has one of these statuses:

| Status | Description |
|---|---|
| `pending` | Waiting to be sent |
| `queued` | Picked up by Celery worker |
| `sent` | Dispatched to WhatsApp API |
| `delivered` | WhatsApp confirmed delivery |
| `read` | Recipient opened the message |
| `failed` | Send failed (see `error_message` / `error_code`) |
| `blocked` | Recipient is on the unsubscribe list |
| `skipped` | Skipped when campaign was cancelled |

## Authentication

All campaign endpoints require the `X-MEGASEND-AUTH` header with your access token.

```
X-MEGASEND-AUTH: YOUR_ACCESS_TOKEN
```

All requests also require an `instance_id` query parameter identifying the WhatsApp instance (phone number) to send from.

## Validate Google Sheets URL

Before creating a campaign from Google Sheets, validate the sheet URL and retrieve its columns. Use this in your campaign wizard to let users map template parameters to sheet columns.

**`POST /campaigns/validate-sheets?instance_id={id}`**

<CodeGroup>

```bash cURL
curl -X POST "https://api.megasend.co.il/campaigns/validate-sheets?instance_id=INSTANCE_ID" \
  -H "X-MEGASEND-AUTH: YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "sheets_url": "https://docs.google.com/spreadsheets/d/SHEET_ID/edit",
    "template_parameters": ["name", "coupon_code"]
  }'
```

```javascript JavaScript
const response = await fetch(
  'https://api.megasend.co.il/campaigns/validate-sheets?instance_id=INSTANCE_ID',
  {
    method: 'POST',
    headers: {
      'X-MEGASEND-AUTH': 'YOUR_ACCESS_TOKEN',
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      sheets_url: 'https://docs.google.com/spreadsheets/d/SHEET_ID/edit',
      template_parameters: ['name', 'coupon_code']
    })
  }
);
const result = await response.json();
```

```python Python
import requests

response = requests.post(
    'https://api.megasend.co.il/campaigns/validate-sheets',
    params={'instance_id': 'INSTANCE_ID'},
    headers={'X-MEGASEND-AUTH': 'YOUR_ACCESS_TOKEN'},
    json={
        'sheets_url': 'https://docs.google.com/spreadsheets/d/SHEET_ID/edit',
        'template_parameters': ['name', 'coupon_code']
    }
)
result = response.json()
```

</CodeGroup>

## Create Campaign

Create a new campaign in `draft` status. Accepts either a Google Sheets audience or a saved Contact Segment — not both.

**`POST /campaigns/?instance_id={id}`**

<CodeGroup>

```bash cURL (Google Sheets)
curl -X POST "https://api.megasend.co.il/campaigns/?instance_id=INSTANCE_ID" \
  -H "X-MEGASEND-AUTH: YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Summer Sale 2026",
    "description": "Promote our summer sale to opted-in customers",
    "template_name": "summer_sale_promo",
    "template_language": "he",
    "template_parameters": ["name", "coupon_code"],
    "sheets_url": "https://docs.google.com/spreadsheets/d/SHEET_ID/edit",
    "column_mapping": {
      "phone": "phone",
      "name": "name",
      "coupon_code": "coupon_code"
    },
    "send_type": "immediate",
    "image_url": "https://cdn.example.com/summer-sale-banner.jpg"
  }'
```

```bash cURL (Segment)
curl -X POST "https://api.megasend.co.il/campaigns/?instance_id=INSTANCE_ID" \
  -H "X-MEGASEND-AUTH: YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "VIP Reactivation",
    "template_name": "vip_reactivation",
    "template_language": "en",
    "template_parameters": ["name"],
    "segment_id": "d9b2e5f3-1234-4abc-9876-000000000001",
    "send_type": "scheduled",
    "scheduled_at": "2026-07-20T10:00:00Z"
  }'
```

</CodeGroup>

### Request Body

<ParamField body="name" type="string" required>
  Campaign display name (1–255 characters).
</ParamField>

<ParamField body="description" type="string">
  Optional description (max 1000 characters).
</ParamField>

<ParamField body="template_name" type="string" required>
  Name of an approved WhatsApp template on this instance. The template must be synced and approved locally — use [Templates](/templates/manage) to sync.
</ParamField>

<ParamField body="template_language" type="string" required>
  Language code of the template (e.g. `he`, `en`, `en_US`). Max 10 characters.
</ParamField>

<ParamField body="template_parameters" type="array of strings" required>
  Ordered list of parameter names that appear in the template body (e.g. `["name", "coupon_code"]`). Must match your `column_mapping` keys.
</ParamField>

<ParamField body="default_parameters" type="object">
  Fallback values for parameters when the sheet cell is empty. Keys are parameter names, values are strings.
</ParamField>

**Audience source — provide exactly one:**

<ParamField body="sheets_url" type="string">
  Google Sheets URL. Required when not using `segment_id`. Must start with `https://docs.google.com/spreadsheets/`.
</ParamField>

<ParamField body="column_mapping" type="object">
  Maps template parameter names (and recipient identifiers) to sheet column names. Required when using `sheets_url`. Must include at least one of `phone` or `bsuid`.

  ```json
  {
    "phone": "Phone Number",
    "name": "Customer Name",
    "coupon_code": "Coupon"
  }
  ```
</ParamField>

<ParamField body="segment_id" type="UUID">
  ID of a saved Contact Segment. Mutually exclusive with `sheets_url` / `column_mapping`.
</ParamField>

**Scheduling:**

<ParamField body="send_type" type="string" default="immediate">
  `immediate` to start now (or after `/start`), or `scheduled` to defer to `scheduled_at`.
</ParamField>

<ParamField body="scheduled_at" type="datetime">
  ISO 8601 UTC datetime for scheduled campaigns. Required when `send_type` is `scheduled`. Must be in the future.
</ParamField>

**Rate and budget control:**

<ParamField body="throttle_rate" type="integer">
  Messages per minute. Auto-optimized by default (10–4800). Override only if you need to stay under a specific rate.
</ParamField>

<ParamField body="batch_size" type="integer">
  Recipients per Celery batch. Auto-optimized by default (50–1000).
</ParamField>

<ParamField body="max_messages" type="integer">
  Budget cap. Campaign stops after this many messages are sent. Min 1.
</ParamField>

**Header media (static — same for all recipients):**

<ParamField body="image_url" type="string">
  Image URL (`https://`) or Meta media ID (numeric string) for IMAGE header templates.
</ParamField>

<ParamField body="video_url" type="string">
  Video URL or Meta media ID for VIDEO header templates.
</ParamField>

<ParamField body="document_url" type="string">
  Document URL or Meta media ID for DOCUMENT header templates.
</ParamField>

**Dynamic per-recipient media (sheets source only):**

<ParamField body="image_url_column" type="string">
  Sheet column name whose value is used as the image URL for each recipient row.
</ParamField>

<ParamField body="video_url_column" type="string">
  Sheet column name for per-recipient video URLs.
</ParamField>

<ParamField body="document_url_column" type="string">
  Sheet column name for per-recipient document URLs.
</ParamField>

<ParamField body="expiration_time_ms" type="integer">
  Unix timestamp in milliseconds for Limited-Time Offer (LTO) template countdown timers.
</ParamField>

## Campaign Lifecycle

### Start Campaign

Start sending a draft campaign immediately, or restart a failed one.

**`POST /campaigns/{campaign_id}/start?instance_id={id}`**

<CodeGroup>

```bash cURL
curl -X POST "https://api.megasend.co.il/campaigns/CAMPAIGN_ID/start?instance_id=INSTANCE_ID" \
  -H "X-MEGASEND-AUTH: YOUR_ACCESS_TOKEN"
```

```javascript JavaScript
const response = await fetch(
  `https://api.megasend.co.il/campaigns/CAMPAIGN_ID/start?instance_id=INSTANCE_ID`,
  { method: 'POST', headers: { 'X-MEGASEND-AUTH': 'YOUR_ACCESS_TOKEN' } }
);
const result = await response.json();
```

```python Python
response = requests.post(
    f'https://api.megasend.co.il/campaigns/CAMPAIGN_ID/start',
    params={'instance_id': 'INSTANCE_ID'},
    headers={'X-MEGASEND-AUTH': 'YOUR_ACCESS_TOKEN'}
)
```

</CodeGroup>

**Query parameters:**

<ParamField query="resume" type="boolean" default="false">
  Pass `resume=true` to continue a `failed` campaign from where it stopped — only unsent recipients are re-processed. Without this flag, all recipients are reset and the campaign starts over.
</ParamField>

**Response:**

```json
{
  "status": "running",
  "campaign_id": "CAMPAIGN_ID",
  "message": "Campaign started successfully"
}
```

### Schedule Campaign

Schedule an existing draft campaign for future execution.

**`POST /campaigns/{campaign_id}/schedule?instance_id={id}`**

Allowed from statuses: `draft`, `completed`, `cancelled`, `failed`. For completed/cancelled/failed campaigns, all recipients are reset to `pending`.

<CodeGroup>

```bash cURL
curl -X POST "https://api.megasend.co.il/campaigns/CAMPAIGN_ID/schedule?instance_id=INSTANCE_ID" \
  -H "X-MEGASEND-AUTH: YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"scheduled_at": "2026-07-20T10:00:00Z"}'
```

```javascript JavaScript
const response = await fetch(
  `https://api.megasend.co.il/campaigns/CAMPAIGN_ID/schedule?instance_id=INSTANCE_ID`,
  {
    method: 'POST',
    headers: {
      'X-MEGASEND-AUTH': 'YOUR_ACCESS_TOKEN',
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({ scheduled_at: '2026-07-20T10:00:00Z' })
  }
);
```

```python Python
response = requests.post(
    f'https://api.megasend.co.il/campaigns/CAMPAIGN_ID/schedule',
```

## Complete Workflow Example

The following end-to-end example shows the full create → start → monitor cycle.

```javascript
const API = 'https://api.megasend.co.il';
const TOKEN = 'YOUR_ACCESS_TOKEN';
const INSTANCE_ID = 'YOUR_INSTANCE_ID';

const headers = {
  'X-MEGASEND-AUTH': TOKEN,
  'Content-Type': 'application/json'
};

// Step 1 — Validate the Google Sheet
const sheetCheck = await fetch(`${API}/campaigns/validate-sheets?instance_id=${INSTANCE_ID}`, {
  method: 'POST',
  headers,
  body: JSON.stringify({
    sheets_url: 'https://docs.google.com/spreadsheets/d/SHEET_ID/edit',
    template_parameters: ['name', 'coupon_code']
  })
}).then(r => r.json());

if (!sheetCheck.valid) {
  throw new Error(`Sheet invalid: ${sheetCheck.error}`);
}
console.log(`Sheet OK — ${sheetCheck.row_count} rows, columns: ${sheetCheck.columns.join(', ')}`);

// Step 2 — Create the campaign
const campaign = await fetch(`${API}/campaigns/?instance_id=${INSTANCE_ID}`, {
  method: 'POST',
  headers,
  body: JSON.stringify({
    name: 'Summer Sale 2026',
    template_name: 'summer_sale_promo',
    template_language: 'he',
    template_parameters: ['name', 'coupon_code'],
    sheets_url: 'https://docs.google.com/spreadsheets/d/SHEET_ID/edit',
    column_mapping: { phone: 'phone', name: 'name', coupon_code: 'coupon_code' },
    send_type: 'immediate',
    image_url: 'https://cdn.example.com/summer-sale-banner.jpg'
  })
}).then(r => r.json());

console.log(`Campaign created: ${campaign.id}`);
console.log(`Valid recipients: ${campaign.total_recipients}`);
if (campaign.invalid_summary?.total_invalid > 0) {
  console.warn(`Excluded: ${JSON.stringify(campaign.invalid_summary)}`);
}

// Step 3 — Send a test message before going live
const testResult = await fetch(`${API}/campaigns/test-send?instance_id=${INSTANCE_ID}`, {
  method: 'POST',
  headers,
  body: JSON.stringify({
    template_name: 'summer_sale_promo',
    template_language: 'he',
    body_params: ['Test User', 'PREVIEW'],
    recipient_phone: '+972501234567'
  })
}).then(r => r.json());
console.log('Test send:', testResult.success ? 'OK' : 'FAILED', testResult.message_id);

// Step 4 — Launch the campaign
const startResult = await fetch(
  `${API}/campaigns/${campaign.id}/start?instance_id=${INSTANCE_ID}`,
  { method: 'POST', headers }
).then(r => r.json());
console.log('Status:', startResult.status); // "running"

// Step 5 — Monitor progress (poll every 30 seconds)
async function monitorCampaign(campaignId) {
  let done = false;
  const terminalStatuses = ['completed', 'cancelled', 'failed'];

  while (!done) {
    await new Promise(resolve => setTimeout(resolve, 30000));

    const latest = await fetch(
      `${API}/campaigns/${campaignId}?instance_id=${INSTANCE_ID}`,
      { headers: { 'X-MEGASEND-AUTH': TOKEN } }
    ).then(r => r.json());

    const { status, messages_sent, total_recipients, messages_delivered, messages_failed } = latest;
    console.log(
      `[${status.toUpperCase()}] sent=${messages_sent}/${total_recipients}` +
      ` delivered=${messages_delivered} failed=${messages_failed}`
    );

    if (latest.auto_paused) {
      console.warn(`Auto-paused: ${latest.pause_reason}`);
    }

    if (terminalStatuses.includes(status)) {
      done = true;
      console.log('Campaign finished with status:', status);
    }
  }
}

await monitorCampaign(campaign.id);

// Step 6 — Inspect failures
const failures = await fetch(
  `${API}/campaigns/${campaign.id}/recipients?instance_id=${INSTANCE_ID}&status=failed&per_page=100`,
  { headers: { 'X-MEGASEND-AUTH': TOKEN } }
).then(r => r.json());

console.log(`Failed recipients: ${failures.total}`);
for (const r of failures.items) {
  console.log(`  ${r.phone_number} — error ${r.error_code}: ${r.error_message}`);
}
```

## State Machine Reference

Valid status transitions:

| From | Allowed next statuses |
|---|---|
| `draft` | `running`, `scheduled`, `cancelled` |
| `scheduled` | `running`, `draft`, `cancelled` |
| `running` | `paused`, `completed`, `cancelled`, `failed` |
| `paused` | `running`, `cancelled` |
| `completed` | `running`, `scheduled` |
| `cancelled` | `running`, `scheduled` |
| `failed` | `running`, `scheduled` |

---

## Error Reference

| HTTP Status | Scenario |
|---|---|
| `400` | Template not approved or not synced; unsupported channel (non-WhatsApp instance); invalid scheduled time; invalid status transition |
| `404` | Campaign or instance not found |
| `429` | Campaign limit reached (total or active per instance); test-send rate limit exceeded |

<Note>
Agent users can access campaign endpoints if they have the `can_manage_campaigns` permission on the instance. Campaign ownership is always attributed to the account owner, not the agent.
</Note>
