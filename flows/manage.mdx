# Automation Flows

The MegaSend Flows API lets you build node-based automation workflows that respond to incoming messages, webhooks, postbacks, or timers. Each flow is a directed graph of nodes connected by edges. When a trigger fires, the engine walks the graph from the single `start` node, executing each node in sequence until it reaches a terminal node or the end of the graph.

## Overview

- **Trigger types** — `incoming_message`, `webhook`, `manual`, `scheduled`, `flow_response`, `postback_instagram`, `postback_messenger`
- **Execution** — flows run asynchronously in the `flows` Celery queue with full execution tracking
- **Multi-platform** — supported node types differ by platform (`whatsapp`, `instagram`, `telnyx`); query `/flows/supported-nodes` before building
- **Conflict detection** — two active flows cannot share the same trigger on the same instance; use `/flows/check-trigger-conflicts` to validate before saving

## Node & Edge Structure

A flow is made up of **nodes** and **edges**.

### Node

| Field | Type | Description |
|---|---|---|
| `id` | string | Unique node identifier within the flow (e.g. `dndnode_1`) |
| `type` | string | Node type (see table below) |
| `position` | `{x: float, y: float}` | Canvas position (for visual editor only) |
| `data` | object | Node-type-specific configuration |

### Supported node types

| Type | Platforms | Description |
|---|---|---|
| `start` | all | Entry point — defines the trigger matching rule |
| `action` | all | Send a plain text message |
| `template_action` | WhatsApp | Send a WhatsApp template message |
| `condition` | all | Branch on a value, contact custom field, or segment membership |
| `delay` | all | Pause for a configured duration then continue. Primary data fields: `delayValue` (int) and `delayType` (`"seconds"` or `"days"`). Also accepts `duration` + `unit` (`"minutes"`, `"hours"`, `"days"`) as a fallback. Clamped to 1 second–7 days. |
| `wait_for_reply` | all | Pause until the contact replies (or timeout) |
| `interactive_buttons` | WhatsApp, Instagram | Send buttons (WhatsApp 1-3 buttons; mapped to Instagram quick replies on IG) |
| `interactive_list` | WhatsApp only | Send an interactive list message |
| `label_action` | all | Assign or unassign contact labels |
| `contact_update` | all | Add a note/update to the contact record |
| `block_action` | all | Block the contact from receiving further messages |
| `cooldown` | all | Terminal node — prevents the contact from re-triggering this flow for a duration |
| `wa_notify` | WhatsApp | Send a template message to a configured recipient (e.g. owner), not the flow contact |
| `ai_response` | all | Generate and send an AI-driven reply |
| `api_request` | all | Make an outbound HTTP request |
| `jump` | all | Jump to another node in the flow |
| `ig_generic_template` | Instagram | Instagram carousel (generic template) |
| `ig_product_template` | Instagram | Instagram product card |
| `ig_media_share` | Instagram | Share an Instagram media object |
| `set_custom_field` | all | Set or clear a contact custom field (key/value pair stored in the contact's custom fields JSONB). Set `clear: true` to remove the key instead of setting it. |

### Edge

| Field | Type | Description |
|---|---|---|
| `id` | string | Unique edge identifier |
| `source` | string | Source node ID |
| `target` | string | Target node ID |
| `sourceHandle` | string (optional) | Handle name on source (e.g. `"true"` / `"false"` for condition branches) |
| `targetHandle` | string (optional) | Handle name on target |

### Minimal flow example

A simple "reply to any message with a welcome text" flow has two nodes and one edge:

```json
{
  "nodes": [
    {
      "id": "start_1",
      "type": "start",
      "position": { "x": 100, "y": 100 },
      "data": { "nodeType": "text_message_start", "type": "any" }
    },
    {
      "id": "action_1",
      "type": "action",
      "position": { "x": 100, "y": 250 },
      "data": { "message": "Welcome! How can we help you today?" }
    }
  ],
  "edges": [
    { "id": "e_start_action", "source": "start_1", "target": "action_1" }
  ]
}
```

## Create Flow

<CodeGroup>

```bash cURL
curl -X POST "https://api.megasend.co.il/flows/" \
  -H "X-MEGASEND-AUTH: YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "instance_id": "123e4567-e89b-12d3-a456-426614174000",
    "name": "Welcome Flow",
    "description": "Greet every new contact",
    "trigger_type": "incoming_message",
    "trigger_config": { "match_type": "any" },
    "is_active": false,
    "priority": 0,
    "nodes": [
      {
        "id": "start_1",
        "type": "start",
        "position": { "x": 100, "y": 100 },
        "data": { "nodeType": "text_message_start", "type": "any" }
      },
      {
        "id": "action_1",
        "type": "action",
        "position": { "x": 100, "y": 250 },
        "data": { "message": "Welcome! How can we help you today?" }
      }
    ],
    "edges": [
      { "id": "e1", "source": "start_1", "target": "action_1" }
    ]
  }'
```

```javascript JavaScript
const response = await fetch('https://api.megasend.co.il/flows/', {
  method: 'POST',
  headers: {
    'X-MEGASEND-AUTH': 'YOUR_ACCESS_TOKEN',

  },
  body: JSON.stringify({
    instance_id: '123e4567-e89b-12d3-a456-426614174000',
    name: 'Welcome Flow',
    trigger_type: 'incoming_message',
    trigger_config: { match_type: 'any' },
    is_active: false,
    priority: 0,
    nodes: [
      {
        id: 'start_1',
        type: 'start',
```

## Update Flow

```bash
PUT /flows/{flow_id}
```

All fields are optional — only send what you want to change. If `nodes` or `edges` are included, the full graph is re-validated (exactly one start node, no orphaned nodes, no circular dependencies).

<CodeGroup>

```bash cURL
curl -X PUT "https://api.megasend.co.il/flows/123e4567-e89b-12d3-a456-426614174000" \
  -H "X-MEGASEND-AUTH: YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Updated Welcome Flow",
    "priority": 5
  }'
```


## Complete Example: Create, Activate, and Monitor a Flow

```python
import requests
import time

BASE = 'https://api.megasend.co.il'
HEADERS = {'X-MEGASEND-AUTH': 'YOUR_ACCESS_TOKEN'}
INSTANCE_ID = '123e4567-e89b-12d3-a456-426614174000'

# 1. Check for trigger conflicts before creating
conflict = requests.post(f'{BASE}/flows/check-trigger-conflicts', headers=HEADERS, json={
    'instance_id': INSTANCE_ID,
    'trigger_type': 'incoming_message',
    'trigger_config': {'match_type': 'contains', 'match_value': 'order'}
}).json()

if conflict['has_conflict']:
    print('Conflict:', conflict['error_message'])
    raise SystemExit(1)

# 2. Create the flow (inactive)
flow = requests.post(f'{BASE}/flows/', headers=HEADERS, json={
    'instance_id': INSTANCE_ID,
    'name': 'Order Status Flow',
    'trigger_type': 'incoming_message',
    'trigger_config': {'match_type': 'contains', 'match_value': 'order'},
    'is_active': False,
    'priority': 0,
    'nodes': [
        {'id': 'start_1', 'type': 'start', 'position': {'x': 100, 'y': 100},
         'data': {'nodeType': 'text_message_start', 'type': 'contains', 'value': 'order'}},
        {'id': 'action_1', 'type': 'action', 'position': {'x': 100, 'y': 250},
         'data': {'message': 'To check your order status, please visit https://example.com/orders'}},
        {'id': 'cooldown_1', 'type': 'cooldown', 'position': {'x': 100, 'y': 400},
         'data': {'duration': 24, 'unit': 'hours'}}
    ],
    'edges': [
        {'id': 'e1', 'source': 'start_1', 'target': 'action_1'},
        {'id': 'e2', 'source': 'action_1', 'target': 'cooldown_1'}
    ]
}).json()

flow_id = flow['id']
print('Created flow:', flow_id)

# 3. Activate the flow
activated = requests.post(f'{BASE}/flows/{flow_id}/activate', headers=HEADERS).json()
print('Active:', activated['is_active'])  # True

# 4. Check stats after some time
stats = requests.get(f'{BASE}/flows/{flow_id}/stats', headers=HEADERS).json()
print(f"Executions: {stats['total_executions']} total, {stats['successful_executions']} successful")

# 5. Inspect execution logs for a failed run
executions = requests.get(
    f'{BASE}/flows/{flow_id}/executions',
    headers=HEADERS,
    params={'page': 1, 'page_size': 5}
).json()

for ex in executions['executions']:
    if ex['status'] == 'failed':
        logs = requests.get(
            f"{BASE}/flows/executions/{ex['id']}/logs",
            headers=HEADERS
        ).json()
        print(f"Failure at node {ex['error_node_id']}: {ex['error_message']}")
        print(f"  {logs['total_logs']} log entries")
```

## Validation Rules

<Note>
These rules are enforced by the API on both create and update. Violating them returns a `422 Unprocessable Entity`.
</Note>

- A flow must have **exactly one** `start` node.
- All nodes must be reachable from the `start` node (no orphaned nodes).
- The flow graph must be acyclic (no circular dependencies that would cause infinite loops).
- `trigger_type` must be one of the allowed values listed in [Create Flow](#create-flow).
- Two active flows on the same instance cannot share the same trigger configuration — use [Check Trigger Conflicts](#check-trigger-conflicts) to detect this before activating.
