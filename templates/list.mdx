Retrieve all WhatsApp Business templates associated with your instances. Templates must be approved by WhatsApp before they can be used to send messages.

## List All Templates

Get all templates for a specific WhatsApp Business instance:

<CodeGroup>

```bash cURL
curl -X GET "https://api.megasend.co.il/templates/{instance_id}/list" \
  -H "X-MEGASEND-AUTH: YOUR_ACCESS_TOKEN"
```

```javascript JavaScript
const response = await fetch('https://api.megasend.co.il/templates/{instance_id}/list', {
  headers: {
    'X-MEGASEND-AUTH': 'YOUR_ACCESS_TOKEN'
  }
});

const templates = await response.json();
console.log(`Found ${templates.data.length} templates`);
```

```python Python
import requests

response = requests.get(
    'https://api.megasend.co.il/templates/{instance_id}/list',
    headers={'X-MEGASEND-AUTH': 'YOUR_ACCESS_TOKEN'}
)

templates = response.json()
print(f"Found {len(templates['data'])} templates")
```

```php PHP
<?php
$response = file_get_contents('https://api.megasend.co.il/templates/{instance_id}/list', false, stream_context_create([
    'http' => [
        'method' => 'GET',
        'header' => 'X-MEGASEND-AUTH: YOUR_ACCESS_TOKEN'
    ]
]));

$templates = json_decode($response, true);
echo "Found " . count($templates['data']) . " templates";
?>
```

</CodeGroup>

## Filter Templates

You can filter templates by status and limit the number returned:

<CodeGroup>

```bash cURL
curl -X GET "https://api.megasend.co.il/templates/{instance_id}/list?status=APPROVED&limit=10" \
  -H "X-MEGASEND-AUTH: YOUR_ACCESS_TOKEN"
```

```javascript JavaScript
const params = new URLSearchParams({
  status: 'APPROVED',
  limit: '10'
});

const response = await fetch(`https://api.megasend.co.il/templates/{instance_id}/list?${params}`, {
  headers: {
    'X-MEGASEND-AUTH': 'YOUR_ACCESS_TOKEN'
  }
});

const templates = await response.json();
```

```python Python
params = {
    'status': 'APPROVED',
    'limit': 10
}

response = requests.get(
    'https://api.megasend.co.il/templates/{instance_id}/list',
    headers={'X-MEGASEND-AUTH': 'YOUR_ACCESS_TOKEN'},
    params=params
)

templates = response.json()
```

</CodeGroup>

## Query Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `status` | string | Filter by template status: `APPROVED`, `PENDING`, `REJECTED` |
| `limit` | integer | Number of templates to return (1–250, default: 250) |

## Response Format

```json
{
  "data": [
    {
      "id": "1234567890",
      "name": "welcome_message",
      "status": "APPROVED",
      "category": "MARKETING",
      "language": "en",
      "components": [
        {
          "type": "BODY",
          "text": "Welcome to {{1}}! Thank you for signing up."
        }
      ],
      "message_send_ttl_seconds": 604800,
      "rejected_reason": null
    }
  ]
}
```

### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Meta template ID |
| `name` | string | Template name |
| `status` | string | Template approval status |
| `category` | string | Template category (e.g. `MARKETING`, `UTILITY`, `AUTHENTICATION`) |
| `language` | string | Template language code (e.g. `en`, `he`) |
| `components` | array | Template component definitions (HEADER, BODY, FOOTER, BUTTONS) |
| `message_send_ttl_seconds` | integer | Time-to-live for sent messages in seconds; `null` if not set |
| `rejected_reason` | string | Reason provided by Meta if the template was rejected; `null` otherwise |

## Template Status Values

- **APPROVED** - Template is approved and can be used for messaging
- **PENDING** - Template is under review by WhatsApp
- **REJECTED** - Template was rejected and cannot be used
- **DISABLED** - Template was disabled by WhatsApp
- **PAUSED** - Template is temporarily paused
- **LIMIT_EXCEEDED** - Template hit quality rating limits

<Note>
Only `APPROVED` templates can be used to send messages. Templates in other statuses are returned for informational purposes.
</Note>

## Error Handling

Common error responses:

```json
{
  "error": {
    "message": "Instance not found or access denied",
    "type": "OAuthException",
    "code": 190
  }
}
```

| Error Code | Description |
|------------|-------------|
| `190` | Invalid access token or instance not found |
| `100` | Invalid parameter value |
| `80004` | Application request limit reached |
