WhatsApp Business templates are pre-approved message formats that enable you to send marketing messages, notifications, and other business communications to customers who haven't messaged you in the last 24 hours.

## Template Structure

WhatsApp templates consist of several components:

- **Header** (optional) - Text, image, video, or document
- **Body** - Main message content with placeholder variables
- **Footer** (optional) - Additional text at the bottom
- **Buttons** (optional) - Call-to-action or quick reply buttons

## Creating Templates

All template endpoints are instance-scoped. Replace `{instance_id}` with your WhatsApp instance UUID.

### Basic Text Template

Create a simple text-only template:

<CodeGroup>

```bash cURL
curl -X POST "https://api.megasend.co.il/templates/{instance_id}/create" \
  -H "X-MEGASEND-AUTH: YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "welcome_message",
    "category": "MARKETING",
    "language": "en",
    "components": [
      {
        "type": "BODY",
        "text": "Welcome to {{1}}! Thank you for signing up. Your account is now active and ready to use."
      }
    ]
  }'
```

```javascript JavaScript
const template = {
  name: 'welcome_message',
  category: 'MARKETING',
  language: 'en',
  components: [
    {
      type: 'BODY',
      text: 'Welcome to {{1}}! Thank you for signing up. Your account is now active and ready to use.'
    }
  ]
};

const response = await fetch('https://api.megasend.co.il/templates/{instance_id}/create', {
  method: 'POST',
  headers: {
    'X-MEGASEND-AUTH': 'YOUR_ACCESS_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify(template)
});

const result = await response.json();
```

```python Python

```

## Template Categories

WhatsApp requires templates to be categorized based on their purpose. Only the following three categories are accepted:

### MARKETING
For promotional content, offers, and product announcements.

**Requirements:**
- Must comply with WhatsApp's marketing policies
- Requires opt-in consent from recipients

**Examples:**
- Product launches
- Sales and discounts
- Newsletter content
- Event invitations

### UTILITY
For account updates, order status, and other service-related messages.

**Requirements:**
- Must provide clear utility to the customer
- Should be relevant to the customer's relationship with your business

**Examples:**
- Account notifications
- Appointment reminders
- Service updates
- Policy changes

### AUTHENTICATION
For verification codes and login confirmations.

**Requirements:**
- Cannot have a HEADER component
- Body text is provided by Meta — do not supply custom body text
- Must use OTP-type buttons
- Automatically approved in most cases

**Examples:**
- OTP codes
- Login confirmations
- Password reset links
- Account verification

## Variable Placeholders

Use `{{1}}`, `{{2}}`, `{{3}}`, etc. as placeholders for dynamic content:

### Body Variables
- Variables must be used in sequential order ({{1}}, {{2}}, {{3}})
- Cannot be empty when sending messages

### Header Variables
- Only one variable allowed in text headers: `{{1}}`
- Media headers don't support variables (media is set per template via asset handle)

### URL Variables
- Dynamic URLs in buttons: `https://example.com/orders/{{1}}`
- Must use HTTPS protocol
- Variable can only be at the end of the URL

## Template Name Rules

Template names may contain **lowercase letters, numbers, and underscores only**. Spaces are automatically converted to underscores. Leading digits are allowed.

**Valid examples:** `welcome_message`, `1st_order_promo`, `order_123_confirm`
**Invalid examples:** `Welcome-Message`, `order.update`, `promo (summer)`

## Response Format

Template creation returns the following response:

```json
{
  "id": "1234567890123456",
  "status": "PENDING",
  "category": "MARKETING"
}
```

### Template Status

- **PENDING** - Submitted for WhatsApp review
- **APPROVED** - Ready to use for sending messages
- **REJECTED** - Not approved, requires modifications
- **PAUSED** - Temporarily paused due to low quality rating
- **DISABLED** - Disabled due to policy violations

## Upload Media for Templates

Image and video header templates require a Meta asset handle obtained via the resumable upload API. Use `upload_type=template` (the default) to get a handle suitable for the `header_handle` field.

<Note>
This endpoint uses Meta's Resumable Upload API, which is different from the media upload used when sending messages. Always use `upload_type=template` when uploading media for template creation.
</Note>

<CodeGroup>

```bash cURL
curl -X POST "https://api.megasend.co.il/templates/{instance_id}/upload-media?upload_type=template" \
  -H "X-MEGASEND-AUTH: YOUR_ACCESS_TOKEN" \
  -F "file=@/path/to/header-image.jpg"
```

```javascript JavaScript
const formData = new FormData();
formData.append('file', document.getElementById('fileInput').files[0]);

const response = await fetch(
  'https://api.megasend.co.il/templates/{instance_id}/upload-media?upload_type=template',
  {
    method: 'POST',
    headers: {
      'X-MEGASEND-AUTH': 'YOUR_ACCESS_TOKEN'
    },
    body: formData
  }
);

const result = await response.json();
// Use result.handle in your template's header_handle field
```

```python Python
import requests

with open('/path/to/header-image.jpg', 'rb') as f:
    response = requests.post(
        'https://api.megasend.co.il/templates/{instance_id}/upload-media',
        params={'upload_type': 'template'},
        headers={'X-MEGASEND-AUTH': 'YOUR_ACCESS_TOKEN'},
        files={'file': f}
    )

result = response.json()
asset_handle = result['handle']  # Use this in header_handle
```

</CodeGroup>

### Upload Response

```json
{
  "success": true,
  "handle": "<ASSET_HANDLE>",
  "id": "<UPLOAD_SESSION_ID>",
  "upload_type": "template",
  "media_type": "image",
  "mime_type": "image/jpeg",
  "filename": "header-image.jpg",
  "size": 204800,
  "message": "Use the 'handle' field in template creation's header_handle parameter"
}
```

Use the `handle` value as the `header_handle` in the template component `example` when creating a template.

### Supported File Types

| Type | Formats | Max Size |
|------|---------|----------|
| Image | JPEG, PNG | 5 MB |
| Video | MP4, 3GPP | 16 MB |
| Document | PDF, DOC, DOCX, XLS, XLSX | 100 MB |

## Template Quality Rating

WhatsApp assigns quality ratings to templates based on user engagement:

### High Quality
- **Benefits**: Higher rate limits, priority delivery
- **Requirements**: Low block rate, high delivery rate, positive user engagement

### Medium Quality
- **Benefits**: Standard rate limits
- **Requirements**: Average engagement metrics

### Low Quality
- **Limitations**: Reduced rate limits, limited delivery
- **Causes**: High block rates, low engagement, user complaints

### Improving Quality

1. **Personalize content** - Use variables for names and relevant information
2. **Provide value** - Ensure messages are useful to recipients
3. **Clear opt-out** - Include unsubscribe instructions
4. **Appropriate timing** - Send messages at reasonable hours
5. **Relevant content** - Match message content to user interests

## Best Practices

### Content Guidelines

1. **Be Clear and Concise**
   - Use simple, direct language
   - Avoid marketing jargon
   - State the purpose clearly

2. **Add Value**
   - Provide useful information
   - Include actionable content
   - Respect the user's time

3. **Follow Policies**
   - No misleading content
   - No prohibited content (adult, illegal, etc.)
   - Respect intellectual property

### Technical Best Practices

1. **Variable Usage**
   - Use variables for personalization
   - Provide meaningful examples
   - Keep variable content short

2. **Button Design**
   - Use clear, action-oriented text
   - Limit to 3 buttons maximum
   - Test URL destinations

3. **Media Guidelines**
   - Use high-quality images
   - Upload via `upload-media` with `upload_type=template` to get a valid asset handle
   - Images: JPEG or PNG, max 5 MB

## Approval Process

### Timeline
- **Authentication templates**: Usually approved within 24 hours
- **Utility templates**: 1-3 business days
- **Marketing templates**: 3-7 business days

### Common Rejection Reasons

1. **Policy Violations**
   - Misleading content
   - Prohibited categories
   - Poor grammar/spelling

2. **Format Issues**
   - Incorrect variable usage
   - Invalid button configurations
   - Unsupported media formats

3. **Quality Issues**
   - Vague or unclear content
   - Too promotional
   - Poor examples

### Resubmission

If a template is rejected:

1. Review the rejection reason
2. Make necessary corrections
3. Resubmit using the create endpoint with the same name (creates a new version)
4. Wait for re-review

## Error Handling

Common template creation errors:

```javascript
try {
  const response = await fetch('https://api.megasend.co.il/templates/{instance_id}/create', {
    method: 'POST',
    headers: {
      'X-MEGASEND-AUTH': 'YOUR_ACCESS_TOKEN',
      'Content-Type': 'application/json'
    },
    body: JSON.stringify(template)
  });

  if (!response.ok) {
    const error = await response.json();

    switch (response.status) {
      case 400:
        console.error('Invalid template format:', error.detail);
        break;
      case 409:
        console.error('Template name already exists:', error.detail);
        break;
      case 422:
        console.error('Template validation failed:', error.detail);
        break;
      default:
        console.error('Unexpected error:', error.detail);
    }
    return;
  }

  const result = await response.json();
  console.log('Template created, Meta ID:', result.id, 'Status:', result.status);

} catch (error) {
  console.error('Network error:', error.message);
}
```

## Common Error Codes

| Code | Message | Solution |
|------|---------|----------|
| 400 | Invalid component format | Check component structure |
| 400 | Invalid upload_type | Use 'template' or 'message' for upload-media |
| 409 | Template name already exists | Use a different name |
| 413 | File too large | Check size limits per media type |
| 415 | Unsupported file type | Use JPEG/PNG for images, MP4/3GPP for video |
| 422 | Invalid variable sequence | Use sequential numbering {{1}}, {{2}} |
| 422 | Unsupported language | Use supported language codes |
| 422 | Invalid category | Use MARKETING, UTILITY, or AUTHENTICATION only |
